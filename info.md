# 角色设定
你是一个资深的 Python 网络开发工程师，精通 TCP 通信、多线程并发处理以及 GUI 开发。

# 任务目标
请帮我开发一款“双人联机五子棋”网络游戏。该项目主要考察 TCP 通信、多线程并发、数据同步以及自定义协议设计。程序需要包含完整的服务器端和客户端，并具备良好的异常处理和程序鲁棒性。

# 系统整体架构
联机五子棋的核心架构是“服务器端维护唯一游戏状态，客户端同步状态并交互”。
1. **TCP 长连接与多线程**：服务器采用“主线程监听连接+子线程处理单个客户端”的模式。客户端使用 TCP 长连接与服务器保持持续通信。
2. **数据一致性**：必须使用互斥锁 (`threading.Lock`) 来避免多客户端同时操作导致的数据混乱。

# 通信协议设计 (自定义应用层协议)
由于 TCP 面向字节流会产生粘包问题，必须严格按照以下自定义协议进行报文的封装与解析：
* **报文头部 (固定 7 字节)**：
  * **魔数 (2 字节)**：固定值 `b'\x58\x51'` (XQ)，用于校验，非法报文直接丢弃。
  * **指令类型 (1 字节)**：
    * `0x01`: 客户端连接 (发送昵称)
    * `0x02`: 落子指令 (发送行、列)
    * `0x03`: 悔棋请求
    * `0x04`: 认输请求
    * `0x05`: 服务器广播 (棋盘状态、当前回合、游戏结果)
    * `0x06`: 心跳指令 (进阶功能)
  * **数据长度 (4 字节)**：后续数据段的字节数，使用大端序 (`int`)，服务器必须按此长度读取后续数据以解决粘包问题。
* **数据段 (可变长度)**：采用 UTF-8 编码。
  * `0x01` 数据示例：“张三”。
  * `0x02` 数据示例：“3,5” (表示行,列)。
  * `0x05` 数据示例：“0,0,0;1,1,1;,黑棋,未分胜负” (棋盘状态,当前回合,游戏结果)。

# 核心功能需求

## 1. 游戏核心逻辑 (简化版)
* **棋盘**：15x15 网格。使用数字标识状态（0:空位置，1:黑棋，2:白棋）。服务器用二维列表存储维护。
* **规则**：黑棋先落子，白棋后落子，交替进行。省略“禁手”等复杂规则，仅保留“五子连珠获胜”和“平局（棋盘下满无胜负）”。
* **胜负判断**：为提升效率，落子后仅判断该棋子所在的“水平、垂直、左上-右下、右上-左下”四个方向是否有连续 5 个同色棋子，无需全盘扫描。

## 2. 服务器端 (Server)
* **基础功能**：
  * 绑定指定端口并监听，最多支持 2 个客户端同时连接。
  * 维护全局唯一棋盘状态、玩家信息及当前回合。
  * 接收并解析客户端协议，校验落子合法性（不能越界、该位置不能有棋子、必须是当前回合玩家）。
  * 合法落子后更新棋盘，并向两个客户端广播最新状态。
  * 异常处理：处理客户端断开、网络中断，清理连接并通知另一方游戏终止。
* **进阶与拔高功能**：
  * **悔棋**：仅允许上一回合玩家悔棋，撤销落子、切换回合并同步。
  * **认输与平局**：处理认输请求和棋盘下满的平局判定，并广播结果。
  * **心跳检测**：超过 15 秒未收到客户端心跳，判定掉线并清理连接。
  * **房间与观战系统**：支持多房间（每房间 2 人）；第三个及以上连接者作为旁观者（仅查看进度，无法落子）。
  * **计分持久化**：记录胜/平/负次数，存入本地 `score.txt` 文件。
  * **超时机制**：玩家落子限时（如 30 秒），超时自动判负。

## 3. 客户端端 (Client)
* **基础界面 (`tkinter` 实现)**：
  * 包含“连接界面”（输入 IP、端口、昵称）和“游戏界面”。
  * 绘制 15x15 棋盘并实现点击网格交互（显示黑/白圆形棋子）。
  * 接收服务器广播，实时且无延迟地同步棋盘和回合提示。
* **进阶与拔高交互**：
  * 非法落子或非当前回合时，使用弹窗提示而非仅文字。
  * 界面显示双方玩家昵称和棋子颜色。
  * 增加落子防抖机制，避免重复请求。
  * **断线重连**：断开后支持重新连接（自动尝试 3 次或手动重新输入），成功后同步当前状态。
  * **游戏设置与记录**：支持调整棋盘/棋子大小、限时时长；游戏结束后可查看落子顺序记录。
  * **快捷键**：实现 `Ctrl+R` 悔棋、`Ctrl+G` 认输。

# 关键避坑指南 (Agent 严格遵守)
1. **彻底解决粘包**：接收数据时，必须先精确接收 7 字节头部，解析出数据长度 `N` 后，再精确接收 `N` 字节的数据段，绝不可一次性 `recv` 大量数据。
2. **数据单一信源**：客户端**绝对不允许**自行修改本地棋盘状态。必须是将落子指令发给服务器，等待服务器广播新状态后，客户端再根据广播进行渲染更新。
3. **GUI 线程安全**：`tkinter` 的 UI 更新必须在主线程进行。网络接收子线程收到数据后，请使用全局变量标记或 `root.after()` 方法通知主线程刷新界面，严禁在子线程直接操作 UI 组件。
4. **编码规范**：所有的字符串传输统一转换为 `UTF-8` 编码 `bytes`，防止中文乱码。
5. **健壮性**：需全覆盖“断线、网络中断、指令错误、越界”等异常捕获，确保程序绝不崩溃。
